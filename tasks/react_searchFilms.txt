1. Створити компонент App:
Цей компонент буде головним і міститиме весь логічний код.
Додай стани для:
Списку фільмів (початково — масив movies, який ми створили).
Пошукового запиту (рядок, що зберігає текст, введений користувачем).
Індикатора завантаження (логічне значення, яке визначає, чи показувати індикатор).
Повідомлення про помилку (рядок для збереження тексту помилки, якщо фільм не знайдено).

2. Створити компонент SearchForm:
Цей компонент буде включати інпут для пошуку та кнопку пошуку.
При введенні тексту користувачем, оновлюй стан пошукового запиту через пропс-функцію onSearchChange, яку передаш із App.
При натисканні кнопки викликай функцію onSearchSubmit, яка також передасться із App, щоб розпочати пошук.
(Використовуємо библиотеку Formik  npm install formik --save)

3. Створити компонент MovieList:
MovieList отримуватиме через пропси відфільтрований масив фільмів та відображатиме його.
Якщо масив порожній, MovieList має перевіряти і відображати компонент ErrorMessage з повідомленням про відсутність результатів.

4. Створити компонент MovieItem:
MovieItem буде окремим компонентом для відображення кожного фільму.
Він отримуватиме дані фільму через пропси (title, year, description, genre, rating).
Стилізуй відображення для кожного елементу так, щоб воно включало всю доступну інформацію.

5. Створити компонент ErrorMessage:
ErrorMessage буде відображати текст помилки, якщо фільми не знайдені.
Цей компонент отримуватиме повідомлення про помилку через пропси.

6. Додати індикатор завантаження:
Створи умовний рендеринг, який показуватиме індикатор (наприклад, текст “Завантаження...”) замість списку фільмів під час виконання пошуку.
Індикатор з’являється при натисканні кнопки пошуку і зникає після отримання результатів.
(Використовуємо спинери звідси  https://mhnpd.github.io/react-loader-spinner/docs/intro/)

7. Додати функціонал пошуку в App:
Створи функцію handleSearchSubmit, яка виконується при натисканні кнопки пошуку.
Спочатку показуй індикатор завантаження (встанови стан для індикатора).
Виконай пошук, фільтруючи масив movies на основі введеного тексту.
Якщо знайдено фільми, онови стан списку фільмів.
Якщо результатів немає, онови стан повідомлення про помилку.
Після завершення пошуку приховай індикатор завантаження.
(Массив фільмів знаходиться у папці assets/movies.js)
(У массиві нема властивості постера,тому можна використовувати по дефолту заглушку  https://via.placeholder.com/200)

Підсумкова структура компонентів:
App — головний компонент, що містить усю логіку.
SearchForm — компонент з інпутом і кнопкою пошуку.
MovieList — компонент, що відображає відфільтрований список фільмів.
MovieItem — компонент для відображення окремого фільму.
ErrorMessage — компонент для відображення помилки, якщо фільм не знайдено.
